# 🚀 Kailash Python SDK v0.3.0 Released!

We're excited to announce the release of Kailash Python SDK v0.3.0, featuring significant architectural improvements that make the SDK more flexible and easier to use.

## 🎯 Release Highlights

### Parameter Lifecycle Architecture
The biggest change in v0.3.0 is how nodes handle parameters. You can now create nodes without specifying all required parameters upfront:

```python
# Before v0.3.0
reader = CSVReaderNode(name="reader", file_path="data.csv")  # All params required

# With v0.3.0
reader = CSVReaderNode(name="reader")  # Create first, configure later!
results = runtime.execute(workflow, parameters={
    "reader": {"file_path": "data.csv"}
})
```

This makes workflows more flexible and enables dynamic parameter values at runtime.

### Centralized Data Management
We've reorganized all data files into a centralized structure with helpful utilities:

```python
from examples.utils.data_paths import get_input_data_path

# Clean, standardized data access
input_file = get_input_data_path("customers.csv")
```

### Enhanced PythonCodeNode
Writing complex Python logic is now much better with full IDE support:

```python
# Define your logic as a proper function
def analyze_data(input_data: list) -> dict:
    """Full syntax highlighting, auto-completion, and debugging!"""
    import pandas as pd
    df = pd.DataFrame(input_data)
    return {"summary": df.describe().to_dict()}

# Create node from function
node = PythonCodeNode.from_function(
    func=analyze_data,
    name="analyzer"
)
```

## 📚 Migration Guide

Most existing code continues to work! The changes maintain backward compatibility, but we recommend gradually adopting the new patterns:

1. **Parameter Flexibility**: Take advantage of runtime parameter passing
2. **Data Paths**: Migrate from hardcoded paths to centralized utilities
3. **PythonCodeNode**: Use `from_function()` for better development experience

See the [full migration guide](https://github.com/terrene-foundation/kailash-py/blob/main/releases/notes/v0.3.0.md) for details.

## 🔧 Breaking Changes

While we've maintained backward compatibility where possible, there are a few breaking changes:

- Runtime modules now call `node.run()` instead of `node.execute()`
- Workflow validation accepts `runtime_parameters` argument
- Data files have been reorganized (old paths still work with warnings)
