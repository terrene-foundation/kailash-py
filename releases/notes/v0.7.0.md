# Kailash SDK v0.7.0 - Detailed Release Notes

**Release Date**: 2025-07-10
**Version**: 0.7.0
**Codename**: "Infrastructure Excellence"

## Executive Summary

Version 0.7.0 marks a major milestone for the Kailash Python SDK with the introduction of complete application frameworks, comprehensive infrastructure improvements, and achievement of 100% E2E test pass rates. This release delivers production-ready solutions for ETL operations (DataFlow), multi-channel orchestration (Nexus), and resolves all infrastructure gaps identified in our systematic E2E audit.

## Major Components

### 1. DataFlow Framework

A complete standalone ETL/database framework with zero-configuration setup and enterprise capabilities.

**Key Features:**
- MongoDB-style query builder for SQL databases
- Redis-based query caching with pattern invalidation
- 4 production example applications
- Comprehensive testing infrastructure
- Docker/Kubernetes deployment ready

**Example:**
```python
from kailash.dataflow import DataFlow

df = DataFlow()
# MongoDB-style queries on SQL databases
active_users = df.users.find({
    "age": {"$gt": 18},
    "status": {"$in": ["active", "premium"]}
}).sort("created_at", -1).limit(100)
```

### 2. Nexus Multi-Channel Platform

Enterprise orchestration platform supporting API, CLI, and MCP interfaces with unified session management.

**Key Features:**
- Single workflow definition, multiple interfaces
- Cross-channel session synchronization
- WebSocket/SSE streaming support
- Enterprise features (multi-tenant, RBAC, marketplace)
- 105 tests with 100% pass rate

**Example:**
```python
from kailash.nexus import create_nexus

nexus = create_nexus(
    title="Enterprise Platform",
    enable_api=True,     # REST API + WebSocket
    enable_cli=True,     # Command-line interface
    enable_mcp=True,     # Model Context Protocol
    channels_synced=True # Cross-channel sync
)

# Register once, access everywhere
nexus.register_workflow("process-data", workflow)
```

## Breaking Changes

1. **MCP Execution Default**
   - AI agents now use `use_real_mcp=True` by default
   - To use mock execution: `use_real_mcp=False`

2. **Gateway Class Renames** (backward compatible)
   - WorkflowAPIGateway → WorkflowServer
   - DurableAPIGateway → DurableWorkflowServer
   - EnhancedDurableAPIGateway → EnterpriseWorkflowServer
