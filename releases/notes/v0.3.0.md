# Kailash Python SDK v0.3.0 Release Notes

**Release Date**: June 10, 2025
**Release Theme**: Parameter Lifecycle Architecture & Data Consolidation

## 🎯 Overview

Version 0.3.0 introduces significant architectural improvements to how the Kailash SDK handles node parameters and data files. These changes make the SDK more flexible and maintainable while resolving long-standing issues with parameter validation and data organization.

## ✨ Major Features

### 1. Parameter Lifecycle Architecture (Session 061/062)

The SDK now supports a clearer separation between node construction, configuration, and execution:

```python
# NEW: Create nodes without required parameters
reader = CSVReaderNode(name="reader")  # Missing file_path is OK!
workflow.add_node("reader", reader)

# Parameters can be provided at runtime
results = runtime.execute(workflow, parameters={
    "reader": {"file_path": "data.csv"}
})
```

**Benefits:**
- More flexible workflow construction
- Better support for dynamic parameter values
- Cleaner separation of concerns
- Improved error messages at the right time

### 2. Centralized Data Management

All data files have been reorganized into a centralized structure with utility functions:

```python
from examples.utils.data_paths import get_input_data_path, get_output_data_path

# NEW: Standardized data access
input_file = get_input_data_path("customers.csv")
output_file = get_output_data_path("results.csv")

# OLD: Hardcoded paths (still works but discouraged)
# input_file = "examples/data/customers.csv"
```

**Structure:**
```
/data/
├── inputs/      # Input data files
├── outputs/     # Generated outputs
├── templates/   # Template files
└── test/        # Test-specific data
```

### 3. Runtime Architecture Fixes

Fixed a critical bug where runtime modules were calling the wrong method:

- ✅ All runtime modules now correctly call `node.run()`
- ✅ Workflow validation supports runtime parameters
- ✅ Direct config updates work properly

### 4. Enhanced PythonCodeNode

Improved support for data science and file processing workflows:

```python
# BEST PRACTICE: Use from_function() for complex logic
def process_data(input_data: list) -> dict:
    """Full IDE support with syntax highlighting and debugging."""
    import pandas as pd
    df = pd.DataFrame(input_data)
    return {"count": len(df), "data": df.to_dict('records')}

node = PythonCodeNode.from_function(
    func=process_data,
    name="processor"
)
```

**Enhancements:**
- Better type inference with `from_function()`
- Expanded module whitelist for data science
- Improved error messages
- Full IDE support for function-based nodes

### 5. Enterprise Workflow Library

Added comprehensive workflow patterns for real-world use:

- Control flow patterns with best practices
- Enterprise data processing workflows
- Industry-specific examples
- Migration from PythonCodeNode-heavy to proper node usage

## 🔄 Breaking Changes

### 1. Runtime Method Change
Runtime modules now call `node.run()` instead of `node.execute()`. This is handled internally, but custom runtime implementations need updating.

### 2. Workflow Validation API
The `validate()` method now accepts runtime parameters:

```python
# NEW
workflow.validate(runtime_parameters={"node1": {"param": "value"}})

# OLD (still works for workflows without runtime params)
workflow.validate()
```

### 3. Data File Organization
Data files have been reorganized. Old paths still work through backward compatibility, but should be migrated:

```python
# Migrate from:
file_path = "examples/data/customers.csv"

# To:
from examples.utils.data_paths import get_input_data_path
file_path = str(get_input_data_path("customers.csv"))
```

## 📚 Migration Guide

### For Most Users

No immediate action required! The changes maintain backward compatibility:

1. **Existing workflows continue to work** - Old patterns are still supported
2. **Gradual migration recommended** - Update to new patterns when convenient
3. **Better error messages** - You'll see clearer errors if parameters are missing

### For Advanced Users

#### Parameter Lifecycle Migration

```python
# OLD: All parameters required at construction
node = CSVReaderNode(name="reader", file_path="data.csv")

# NEW: Flexible parameter provision
node = CSVReaderNode(name="reader")
# Configure later or at runtime
```

#### Data Path Migration

```python
# Simple migration script
import os
from examples.utils.data_paths import get_input_data_path

# Update your file paths
old_path = "examples/data/myfile.csv"
new_path = str(get_input_data_path("myfile.csv"))
```

#### PythonCodeNode Best Practices

For code blocks longer than 3 lines, migrate to `from_function()`:

```python
# OLD: String-based code
node = PythonCodeNode(
    name="processor",
    code="""
import pandas as pd
df = pd.DataFrame(data)
# ... many lines of code ...
result = {"output": df.to_dict()}
"""
)

# NEW: Function-based with IDE support
def process_dataframe(data: list) -> dict:
    import pandas as pd
    df = pd.DataFrame(data)
    # ... full IDE support! ...
    return {"output": df.to_dict()}

node = PythonCodeNode.from_function(
    func=process_dataframe,
    name="processor"
)
```
