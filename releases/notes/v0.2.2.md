# Kailash Python SDK v0.2.2 Release Notes

**Release Date**: June 10, 2025

## 🎯 Overview

Version 0.2.2 focuses on documentation improvements and test modernization. This release introduces a major restructuring of the documentation to better serve both SDK users and contributors, along with a comprehensive migration to the new CycleBuilder API.

## 🚀 Key Features

### 1. Documentation Restructuring
- **Reorganized for clarity**: Split documentation into `sdk-users/` and `# contrib (removed)/` directories
- **Production-ready workflows**: New workflow library with industry-specific examples
- **Improved navigation**: CLAUDE.md files at each level for efficient AI-assisted development
- **Better user experience**: Clear separation between building WITH the SDK vs developing the SDK

### 2. CycleBuilder API Migration
- **130+ test updates**: Migrated deprecated `workflow.connect(..., cycle=True)` to new fluent API
- **Improved readability**: Method chaining for intuitive cycle configuration
- **Backward compatibility**: Complex conditional cycles still supported with old API

### 3. Enhanced Workflow Library
- **By Pattern**: API integration, data processing, file processing workflows
- **By Industry**: Healthcare, finance-specific implementations
- **By Enterprise**: Customer operations, multi-system integrations
- **Quick Start**: 30-second workflow patterns for rapid prototyping

## 🐛 Fixes

- Fixed conditional cycle test failures with SwitchNode parameter passing
- Resolved formatting inconsistencies with black and isort
- Updated Sphinx documentation to eliminate broken references
- Fixed test patterns to demonstrate CycleBuilder best practices

## 💻 Code Examples

### Old Cycle API (Deprecated):
```python
workflow.connect(
    "processor",
    "processor",
    cycle=True,
    max_iterations=10,
    convergence_check="converged == True"
)
```

### New CycleBuilder API:
```python
workflow.create_cycle("optimization_loop").connect(
    "processor",
    "processor"
).max_iterations(10).converge_when("converged == True").build()
```

## 🔄 Migration Guide

For users with existing cyclic workflows:
1. The old API still works but shows deprecation warnings
2. Use CycleBuilder for new workflows
3. For conditional cycles with SwitchNode, continue using old API until CycleBuilder supports conditions
