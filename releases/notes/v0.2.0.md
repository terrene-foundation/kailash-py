# Kailash Python SDK v0.2.0 Release Notes

**Release Date:** June 8, 2025

## 🎉 Major Release: Universal Hybrid Cyclic Graph Architecture

We're excited to announce the release of Kailash Python SDK v0.2.0, featuring the complete implementation of our Universal Hybrid Cyclic Graph Architecture. This release represents a significant milestone in workflow automation, enabling iterative, feedback-driven, and self-improving workflows.

## 🚀 Key Features

### Universal Hybrid Cyclic Graph Architecture

The centerpiece of v0.2.0 is the complete Phase 6 implementation of cyclic workflows:

- **CycleAwareNode Base Class**: A powerful foundation for building iterative nodes with built-in helper methods
- **Native Cycle Support**: First-class support for cycles in workflow graphs
- **High Performance**: Optimized execution achieving ~30,000 iterations/second
- **Production Ready**: Battle-tested with workflows handling 100,000+ iterations

### Developer Tools & Analytics

We've added comprehensive developer tools to make working with cyclic workflows a breeze:

- **CycleAnalyzer**: Deep performance analysis of cyclic workflows
- **CycleDebugger**: Interactive debugging capabilities
- **CycleProfiler**: Detailed performance profiling
- **CycleBuilder**: Fluent API for easy cycle construction
- **Type-Safe Configuration**: Strong typing for cycle configurations

### Enhanced Data Science Support

- Improved PythonCodeNode with better DataFrame and NumPy compatibility
- Automatic serialization for complex data types
- Platform-specific type compatibility checks
- Memory-efficient data processing patterns

## 📝 Breaking Changes

None! This release maintains full backward compatibility with v0.1.x.

## 💡 Quick Example

```python
from kailash import Workflow
from kailash.nodes.code import PythonCodeNode
from kailash.runtime import LocalRuntime

# Create a simple iterative workflow
workflow = Workflow(name="iterative_optimizer")

# Define nodes
optimizer = PythonCodeNode(
    name="optimizer",
    code="""
import numpy as np
# Access previous iteration's state
prev_value = cycle_info.get('node_state', {}).get('value', 10.0)
# Gradient descent step
value = prev_value - 0.1 * (2 * prev_value)
converged = abs(value) < 0.01
result = {'value': value, 'converged': converged}
"""
)

# Add node and create cycle
workflow.add_node(optimizer)
workflow.connect(
    optimizer, optimizer,
    cycle=True,
    max_iterations=100,
    convergence_check="converged == True",
    mapping={"value": "value"}
)

# Execute the cyclic workflow
runtime = LocalRuntime()
results = runtime.execute(workflow)
print(f"Converged to: {results['optimizer']['value']}")
```

## 🎯 Use Cases

The cyclic architecture enables powerful new patterns:
