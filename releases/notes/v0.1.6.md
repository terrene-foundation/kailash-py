# Kailash Python SDK v0.1.6 Release Notes

**Release Date**: June 5, 2025
**Version**: 0.1.6
**Type**: Minor Release

## 🎯 Release Overview

This release introduces the Kailash Workflow Studio backend infrastructure, a comprehensive JWT authentication and access control system, and significant documentation improvements. This marks a major step towards enterprise-ready workflow management with visual building capabilities.

## ✨ Key Features

### 🏗️ Workflow Studio Backend Infrastructure

**Complete REST API for Visual Workflow Building**
- Full CRUD operations for workflows with version control
- Node discovery API with category-based organization
- Real-time workflow execution with WebSocket status updates
- Custom node creation and management system
- Workflow import/export functionality
- Docker-based infrastructure (PostgreSQL, Redis, MinIO)

### 🔐 JWT Authentication & Multi-Tenancy

**Enterprise-Grade Authentication System**
- JWT-based authentication with access/refresh token pattern
- Multi-tenant architecture with complete isolation
- User registration and secure login system
- API key authentication for service accounts
- Token expiration and refresh handling
- Password hashing with bcrypt

### 🛡️ Comprehensive Access Control

**Fine-Grained Permission Management**
- Role-Based Access Control (RBAC) with Admin, Editor, Viewer roles
- Node-level permissions (execute, read_output, write_input)
- Workflow-level permissions (view, execute, modify, delete, share)
- Field-level data masking for sensitive information
- Permission-based conditional routing
- Access-controlled runtime with transparent security layer

**Complete Documentation Compliance**

## 💡 Usage Examples

### JWT Authentication
```python
from kailash.api.auth import JWTAuth

auth = JWTAuth(secret_key="your-secret-key")
token = auth.create_access_token(user_id="user123", tenant_id="tenant1")
```

### Access Control
```python
from kailash.access_control import UserContext, get_access_control_manager
from kailash.runtime.access_controlled import AccessControlledRuntime

user = UserContext(
    user_id="john_doe",
    tenant_id="acme_corp",
    email="john@acme.com",
    roles=["analyst"]
)

runtime = AccessControlledRuntime(user_context=user)
result, run_id = runtime.execute(workflow)
```

### Workflow Studio API
```python
from kailash.api.studio import WorkflowStudioAPI

api = WorkflowStudioAPI()
api.run(port=8080)
```

## 🔄 Migration Guide

### From v0.1.5 to v0.1.6

1. **DateTime Usage**: Update any `datetime.utcnow()` calls to `datetime.now(timezone.utc)`
2. **Access Control**: Optionally enable access control by setting `acm.enabled = True`
3. **Workflow Studio**: New feature, no migration needed
