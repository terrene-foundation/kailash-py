# v0.6.4 Release Notes

## Release Date: 2025-07-06

## Overview

This release focuses on enterprise-grade parameter injection capabilities, comprehensive E2E test improvements achieving 100% pass rate, and significant documentation enhancements based on test findings.

## Key Features

### 1. Enterprise Parameter Injection System
- **WorkflowBuilder Parameter Mapping**: New `add_workflow_inputs()` method for explicit parameter mapping
- **Dot Notation Support**: Access nested parameters using paths like `data.user_id` or `settings.processing.mode`
- **Direct Namespace Injection**: PythonCodeNode parameters are now correctly injected directly into the execution namespace

### 2. E2E Test Excellence
- **100% Pass Rate**: All 4 comprehensive E2E tests now passing consistently
- **Test Stability**: Fixed transaction isolation issues and improved test reliability
- **Enterprise RBAC Testing**: Complete end-to-end testing of role-based access control

### 3. Documentation Quality Improvements
- **E2E Test Findings**: Comprehensive documentation of patterns discovered during testing
- **Corrected Examples**: Fixed permission check structures, parameter patterns, and execution methods
- **Best Practices**: Added common gotchas and recommended patterns based on real-world testing

## Bug Fixes

1. **Permission Check Result Structure**
   - Fixed nested result structure: `result.check.allowed` (not `result.allowed`)
   - Updated all documentation and examples with correct pattern

2. **PythonCodeNode Parameter Access**
   - Parameters now correctly injected directly into namespace
   - Fixed `input_data is not defined` errors

3. **Integration Test Stability**
   - Improved cache handling in integration tests
   - Fixed async handling in DeferredConfigNode
   - Added missing nodes to test configurations

4. **Role Management**
   - Documented automatic role ID generation from role names
   - Fixed transaction isolation issues with direct node execution

## Technical Improvements

### Parameter Injection
```python
# New WorkflowBuilder parameter mapping
workflow_builder.add_workflow_inputs("processor", {
    "user_id": "data.user_id",      # Dot notation support
    "config": "settings.processing"  # Deep nesting supported
})

# PythonCodeNode direct access
workflow_builder.add_node("PythonCodeNode", "validator", {
    "code": """
# Parameters are injected directly (not via input_data)
if not enterprise or not tenant_id:
    raise ValueError("Missing parameters")
"""
})
```

### Permission Checking
```python
# Correct permission check pattern
if result.get("result", {}).get("check", {}).get("allowed", False):
    print("Permission granted!")
```

### Direct Node Execution
```python
# Preferred for database operations
role_node = RoleManagementNode(
    operation="create_role",
    tenant_id=tenant_id,
    database_config=db_config
)
result = role_node.execute(role_data=data)
```

## Breaking Changes

None - This release is fully backward compatible.

## Migration Guide

If you're using PythonCodeNode with `input_data.get()` patterns:
1. Remove `input_data.get()` calls
2. Access parameters directly from the namespace
3. Connection data still comes through mapped parameter names
