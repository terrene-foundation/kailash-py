# Kailash Python SDK v0.6.6 Release Notes

**Release Date**: TBD
**Type**: Bug Fix & API Standardization

## Overview

Version 0.6.6 fixes a critical bug in AgentUIMiddleware that prevented shared workflows from being executed across sessions. This release also standardizes the API by deprecating `execute_workflow()` in favor of `execute()`.

## Key Improvements

### 🐛 Critical Bug Fix: Shared Workflow Execution

**Problem**: Workflows registered with `make_shared=True` would fail when executed from any session with "Workflow not found in session" error.

**Solution**: Shared workflows are now automatically copied to sessions when first executed, enabling proper multi-tenant patterns.

**Impact**:
- Removes need for manual workarounds
- Enables enterprise-scale workflow sharing
- Improves developer experience

### 🔧 API Standardization

**Change**: `execute_workflow()` deprecated in favor of `execute()`

**Reason**: Consistency with runtime API patterns across the SDK

**Migration**: Simple method rename, all parameters remain the same

## Technical Details

### Shared Workflow Fix
- Modified `AgentUIMiddleware.execute()` to handle shared workflows
- Maintains session isolation while enabling workflow reuse
- Session-specific workflows take priority over shared ones

### Test Coverage
- 4 new integration tests validate shared workflow behavior
- 100% backward compatibility maintained
- All 79 middleware tests passing

## Migration Guide

### For Shared Workflows
```python
# Before (with workaround)
session = await middleware.get_session(session_id)
if "workflow" not in session.workflows:
    session.add_workflow("workflow", middleware.shared_workflows["workflow"])

# After (automatic)
# Just execute directly - no workaround needed!
await middleware.execute(session_id, "workflow", inputs)
```

### For API Methods
```python
# Before
await middleware.execute_workflow(session_id, workflow_id, inputs)

# After
await middleware.execute(session_id, workflow_id, inputs)
```
