## Release Process

### Build & Package
- [ ] Clean build directory: `rm -rf dist/ build/`
- [ ] Build distribution: `python -m build`
- [ ] Check distribution: `twine check dist/*`
- [ ] Test installation from wheel in clean environment

### Git Operations
- [ ] Commit all changes to feature branch
- [ ] Create PR to main branch
- [ ] PR approved by at least one reviewer
- [ ] Merge PR to main
- [ ] Create git tag: `git tag -a v0.2.0 -m "Release version 0.2.0"`
- [ ] Push tag: `git push origin v0.2.0`

### PyPI Release
- [ ] Upload to TestPyPI: `twine upload --repository testpypi dist/*`
- [ ] Test installation from TestPyPI
- [ ] Upload to PyPI: `twine upload dist/*`
- [ ] Verify installation: `pip install kailash==0.2.0`

### GitHub Release
- [ ] Create GitHub release from tag
- [ ] Attach release notes
- [ ] Attach wheel and source distribution
- [ ] Mark as latest release

If issues are discovered:

3. Fix issues and create v0.2.1
