## Release Notes Highlights

### Major Features
1. **Parameter Lifecycle Architecture**
   - Nodes can be created without required parameters
   - Parameters validated at execution time
   - More flexible workflow construction

2. **Data Consolidation**
   - Centralized `/data/` structure
   - Standardized data access utilities
   - Backward compatibility maintained

3. **Runtime Architecture Fixes**
   - Fixed critical `execute()` → `run()` bug
   - Improved workflow validation
   - Better error messages

4. **PythonCodeNode Enhancements**
   - Better `from_function()` support
   - Enhanced data science capabilities
   - Improved type inference

5. **Enterprise Workflow Library**
   - Real-world workflow patterns
   - Best practices documentation
   - Production-ready examples

### Breaking Changes
- Runtime modules now call `node.run()` instead of `node.execute()`
- Workflow validation now accepts `runtime_parameters`
- Data files reorganized (backward compatible)

### Migration Guide Available
See `releases/notes/v0.3.0.md` for detailed migration instructions.
