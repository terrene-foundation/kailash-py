# PR: Fix AgentUIMiddleware Shared Workflow Execution

## Summary

This PR fixes a critical bug in AgentUIMiddleware where shared workflows couldn't be executed from sessions. The fix enables proper multi-tenant workflow patterns without manual workarounds.

## Changes

### Bug Fix
- **Fixed**: AgentUIMiddleware now automatically copies shared workflows to sessions when first executed
- **File**: `src/kailash/middleware/core/agent_ui.py` (lines 656-666)

### API Standardization
- **Updated**: All documentation to use `execute()` instead of deprecated `execute_workflow()`
- **Files**:
  - `sdk-users/middleware/agent-ui-communication.md` (6 occurrences updated)
  - Added shared workflow behavior section

### Documentation
- **Created**: Migration guide `v0.6.5-shared-workflow-fix.md`
- **Updated**: Version-specific migration index
- **Added**: Comprehensive explanation of shared workflow behavior

### Testing
- **Added**: `test_agent_ui_shared_workflow_fix.py` with 4 integration tests
- **Coverage**: Session isolation, priority handling, error cases
- **Result**: All 79 middleware tests passing

## Technical Details

### Before
```python
# This would fail with "Workflow not found in session"
await middleware.register_workflow("auth", workflow, make_shared=True)
await middleware.execute(session_id, "auth", inputs)  # ❌ ERROR
```

### After
```python
# Now works automatically
await middleware.register_workflow("auth", workflow, make_shared=True)
await middleware.execute(session_id, "auth", inputs)  # ✅ SUCCESS
```
