## What shipped

Compute **one shared-cache URI per DataFlow instance** (`file:df_mem_<id>?mode=memory&cache=shared`,
`_memory_db_uri`) + a **lifetime anchor connection**, injected at every SQLite
connection-construction site: the cached async CRUD node, all 5 `SyncDDLExecutor`
sites, the migration system, `migration_connection_manager`, `migration_test_framework`,
the model registry (`_registry_connection_url`), the schema-state manager
(`_state_connection_url`), and the `_get_async_sql_connection` fallback DDL path.
`config.database.url` stays the literal `:memory:` so the ~12 dialect-detection branches
are unchanged; the rewrite fires **only** for bare `:memory:` — file-backed SQLite /
PostgreSQL / MySQL are byte-for-byte unaffected.

- **kailash 2.45.3**: `SQLDatabaseNode` uses `StaticPool` + `check_same_thread=False` for a
  SQLite memory connection string (fixes the sync-registry `sqlite3.ProgrammingError:
SQLite objects created in a thread ...`), normalizing `file:` shared-cache →
  `sqlite:///file:...&uri=true`. New `dispose_pools_for(connection_string)` (targeted,
  per-URI pool dispose).
- **kailash-dataflow 2.13.6**: the shared-cache URI + anchor wiring above; `close()` /
  `close_async()` dispose the registry `StaticPool` for the instance's URI (prevents a
  leaked in-memory DB + `id()`-reuse cross-instance aliasing).
