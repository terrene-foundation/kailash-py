## What shipped

- **#1218 (kailash 2.28.4, patch)** — `WorkflowAPI` gateway `/workflows/{name}/execute` (`src/kailash/api/workflow_api.py`) now honors a typed HTTP status raised by a workflow node instead of collapsing every exception to a generic 500. `_find_typed_status_exc` walks the `__cause__`/`__context__` chain (cycle-guarded; the async runtime wraps node exceptions in `WorkflowExecutionError(...) from e`) and maps `status_code` + `body` to the response only when BOTH are present (the `NexusHandlerError` contract). Genuine internal errors still collapse to the canonical `{"detail": "Internal server error"}` 500, raw error logged server-side only. The `body`-presence gate (security-reviewer LOW) prevents a stray `HTTPException(404, detail)` from echoing `str(exc)`.
- **#1216 (kailash-nexus 2.9.0, minor)** — WebSocket handshake auth resolves in a `serve(process_request=...)` callback BEFORE the Upgrade; unauthenticated handshake → pre-upgrade HTTP 401/403 + JSON envelope (was post-upgrade WS-1008). Fail-closed at two layers. Rejection body never echoes credentials.
- **#1217 (kailash-nexus 2.9.0, minor)** — `serve(select_subprotocol=...)` echoes the accepted subprotocol via `Sec-WebSocket-Protocol` (RFC 6455 §4.2.2); reject-only allowlist preserved (unlisted → WS-1002), echo is allowlist-only.
- nexus floor bumped to `kailash>=2.28.4` so the #1218 typed-status fix ships with the WS features.

Single shared `serve()`-callback rewire of `transports/websocket.py::_serve`/`_connection_handler` addresses #1216 + #1217; handshake-auth moved fully pre-upgrade, redundant post-upgrade resolver removed (orphan-detection Rule 4).
