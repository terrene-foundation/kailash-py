## Changes

### Cluster A — BaseTool override conformance (Shard 1, PR #818)

22 `def execute(...)` override sites across 9 files in
`packages/kailash-kaizen/src/kaizen/tools/native/` widened from narrower
typed signatures to the canonical
`*, <named keyword-only params>, **_kwargs: Any`
pattern. Pyright `reportIncompatibleMethodOverride` cleared.

Underscore-prefix sink documents intent ("intentionally accepted but
not consumed in this body" — LSP override conformance, not silent
fallback). LLM tool-calling surface (`BaseTool.get_schema()`) is
unaffected.

Initial PR shipped 19 of 22 sites uniform; reviewer flagged 3 stragglers
(`skill_tool.py:101`, `task_tool.py:127`, `todo_tool.py:282`). Fix-up
commit `0c918963` landed same-session per
`rules/autonomous-execution.md` Rule 4 + `rules/git.md` § "Commit-message
claim accuracy" follow-up rule.

### Cluster B — Optional/None safety (Shard 1, PR #818)

6 root-cause clusters across 12 pyright issues in
`tools/native/notebook_tool.py`, `tools/native/task_tool.py`,
`tools/native/interaction_tool.py`, `research/parser.py`. Typed guards
(`Rule 3a`-style `_require_X()` helpers), local-variable narrowing for
validator-checked paths, lazy-import sentinel typing.

### Cluster C1 — Vestigial subsystem deleted (Shard 2, PR #820)

PR #75 (commit `801de2bb`, 2026-03-25) moved the experimental
research-pattern subsystem from `kaizen.research/` to
`kaizen-agents/research_patterns/` but left behind a vestigial cluster:
5 src files + 7 dangling `__all__` re-exports + 8 dead-path test files

- 1 examples directory. All 5 src files carried unguarded
  `from kaizen_agents.research_patterns.experimental import ExperimentalFeature`
  imports — `kaizen-agents` is NOT in `kailash-kaizen` deps; clean
  `pip install kailash-kaizen` raised `ModuleNotFoundError` at first
  `from kaizen.research import …`.

The original `/todos` plan flagged only `feature_manager.py` for
deletion; mid-implement discovery surfaced 4 additional sites with the
same unguarded import. Per `rules/autonomous-execution.md` Rule 4
(Fix-Immediately when same-class gap fits within shard budget) + human
approval, expanded the deletion to the entire vestigial cluster
(5,019 lines deleted across 14 files).

Per `rules/zero-tolerance.md` Rule 6a — no `DeprecationWarning` shim
owed because the removed surface raised `ModuleNotFoundError` on first
use since 2026-03-25 (verified via `git log --oneline --since=2026-03-25`
on the deleted source paths returning zero results). CHANGELOG migration
entry covers callers who should migrate to `kaizen_agents.research_patterns.*`.

### Cluster C2 — Optional extras + bs4 loud failure (Shard 2, PR #820)

`pyproject.toml` declares `[research]` (`arxiv>=2.0`, `pypdf>=4.0`) and
`[web-search]` (`duckduckgo-search>=6.0`, `beautifulsoup4>=4.12`)
optional-extras groups per `rules/dependencies.md` "Declared = Imported".

`WebFetchTool._extract_text` converted from silent-degradation (logged
`WARNING`, returned raw HTML) to loud failure (`ImportError` with
install hint, caught at call site, returned as `NativeToolResult.from_error`
to LLM). Module-scope `_BeautifulSoup` sentinel replaces inline
`try/except ImportError` per `rules/dependencies.md` BLOCKED
anti-patterns.

### Cluster D — `ResearchAdapter` runtime bug (Shard 1, PR #818)

`ResearchAdapter.create_signature_adapter` was passing `dict` to
`Signature(inputs=, outputs=)` which expects `List[str]` — silently
corrupting `Signature._inputs_list` since the file was authored.
Behavioral regression at
`tests/regression/test_issue_814_research_adapter_inputs_list.py`.
