## #1352 — PEP 563 Signatures silently empty structured output (HIGH)

A Kaizen `Signature` defined in a module using `from __future__ import annotations`
(PEP 563) stored each output field's type as the annotation **string** (`'str'`,
`'List[dict]'`) instead of a type object. `JSONOutputParser._convert_to_type` then
ran `isinstance(value, 'str')` → `TypeError: isinstance() arg 2 must be a type ...`,
swallowed by the parser's own `except (json.JSONDecodeError, TypeError)` — silently
degrading a valid JSON response to `{}`.

Fix (two layers):

- **Root cause** — `SignatureMeta` resolves string annotations to real type objects
  after class creation via `typing.get_type_hints` (the issue-reporter-suggested fix),
  resolving against the class's own module globals. Stored field types are identical
  with or without PEP 563. Best-effort: unresolvable forward refs keep string form and
  the parser guard handles them — construction never raises.
- **Defense-in-depth** — `JSONOutputParser._convert_to_type` returns the value
  unchanged if an annotation string ever survives construction (dynamically-`exec`'d
  Signatures, unresolvable forward refs) instead of raising.

PR #1357 (`fix/issue-1352-pep563-signature-parser`, merge `c5f544e30`, change commit
`662bdc182`). Regression: `tests/regression/test_issue_1352_pep563_signature_parser.py`
(6 tests). Cross-SDK: PEP 563 is Python-intrinsic — no kailash-rs equivalent.

## #1351 — Gemini adapters on deprecated google.generativeai SDK (MEDIUM)

The Gemini adapters imported the end-of-life `google.generativeai` package, declared
only in `[dev]` — so a normal `pip install kaizen-agents` raised `ImportError` on the
Gemini path (and emitted a `FutureWarning` when present).

Fix:

- Migrated `delegate/adapters/google_adapter.py` + `runtime_adapters/gemini_cli.py` to
  the supported `google.genai` SDK (`genai.Client` +
  `client.aio.models.generate_content_stream` / `client.models.generate_content` +
  `types.GenerateContentConfig` / `types.Tool` / `types.ToolCodeExecution`; generation
  knobs moved into the per-request config; streaming chunks read via
  `candidates[0].content.parts`). Also fixed the packaged developer doc teaching the
  removed `_generative_model` API.
- Declared `google-genai` AND the sibling `anthropic` (identical undeclared-runtime
  defect — `anthropic_adapter.py` + `orchestration/adapters.py` import it at runtime)
  in `[project.dependencies]`; removed the deprecated `google-generativeai` pin.

PR #1358 (`fix/issue-1351-google-genai-migration`, merge `858a9af8a`, change commit
`909ed5470`). Regression: `tests/regression/test_issue_1351_google_genai_migration.py`
(6 tests). The 2 Anthropic adapter tests that previously failed on a no-`[dev]` venv now
pass with the runtime dep.
