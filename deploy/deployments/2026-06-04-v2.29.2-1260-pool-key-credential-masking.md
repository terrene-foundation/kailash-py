## What shipped

### Connection-pool keys no longer leak credentials at any observable surface (#1260)

`AsyncSQLDatabaseNode` pool keys embed a raw connection string in their third
`|`-segment (per `_generate_pool_key`: `loop_id|db_type|connection_string|min|max`);
Redis pool keys are `redis://:pass@host/dbN`. The connection-string segment can carry
`user:password@` credentials. Multiple surfaces interpolated the **full key**:

- **Log lines** — disposal (`clear_shared_pools`), cleanup (`_cleanup_closed_loop_pools`),
  the per-pool lock manager, the idle-pool reaper, and the fallback-pool path, several at
  **WARN/ERROR** level (ships to log aggregators with broader access than the DB).
- **Prometheus metric labels** — `kailash.monitoring.asyncsql_metrics` used the raw key as
  a `pool_key` label value (same aggregator exposure + an unbounded-cardinality hazard).
- **`PoolExhaustedError`** — message + `.pool_key` attribute.
- **Public diagnostic return values** — `AsyncSQLDatabaseNode.get_pool_info` /
  `get_pool_metrics` / `pool_keys` / `get_lock_metrics`, and `RedisPoolManagerNode`
  pool-status / health-report / exec-result / cleanup-list return surfaces.

**Fix:** a new shared helper `kailash.utils.url_credentials.redact_pool_key` masks **only**
the credential-bearing connection-URL segment via the canonical `mask_url`
(`postgresql://***@host`), preserving the loop-id / db-type / pool-size segments (and the
Redis db index) for forensic correlation. Redaction is **deterministic**, so log/metric
correlation by string equality still works AND Prometheus label cardinality is now bounded
(`tenant-isolation.md` Rule 4). The helper reconstructs the connection-string field from the
middle `|`-segments (`"|".join(parts[2:-2])`) so a literal `|` inside a password cannot leak
the password tail. Every log line, metric label, exception, and diagnostic return value across
`async_sql.py`, `exceptions.py`, `redis_pool_manager.py`, and `asyncsql_metrics.py` routes
through the helper. The credential-free `host:port:db:user` fallback form (no `://`) is left
unmasked — it carries no password and host context is useful.

Severity LOW-MEDIUM: gated to disposal / error / diagnostic paths, and connection strings
often use env-var / `.pgpass` passwords — but WARN/ERROR lines and metric labels reach
aggregators. Pre-existing.
