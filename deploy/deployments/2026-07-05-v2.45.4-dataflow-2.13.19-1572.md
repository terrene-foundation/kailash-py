# Deployment — kailash 2.45.4 + kailash-dataflow 2.13.19 (#1572)

**Date:** 2026-07-05
**Type:** core-first two-package patch release
**Issue:** #1572 (MySQL/PostgreSQL adapter pools leaked on DataFlow's transient sync→async bridge loop → GC-time `RuntimeError: Event loop is closed` + `ResourceWarning: Unclosed connection` after `await db.close_async()`)

## What shipped

- **kailash 2.45.3 → 2.45.4** — new `kailash.utils.loop_pool_registry` (per-loop transient-bridge pool-drain registry, marker `_kailash_transient_bridge_loop`, 5s/pool bound, `EnterpriseConnectionPool` covered transitively via its inner adapter's `connect()`); `AsyncSQL` PostgreSQL/MySQL adapter `connect()` registers its pool drain; `disconnect()` made idempotent.
- **kailash-dataflow 2.13.18 → 2.13.19** — the bridge (`dataflow.core.async_utils._run_on_new_loop`) drains registered pools before closing its transient loop; teardown order drain → cancel_all_tasks → shutdown_asyncgens → shutdown_default_executor → close → detach; both bridge branches (thread-pool + no-running-loop). DataFlow `MySQL`/`PostgreSQL` adapters register their pool-close at pool creation. Floor bumped `kailash>=2.45.3 → kailash>=2.45.4`.

**Core-first rationale:** the fix's registry lives in core `kailash` and the DataFlow adapters import it; a dataflow-only release would `ImportError`, so core published first with the floor bump.
