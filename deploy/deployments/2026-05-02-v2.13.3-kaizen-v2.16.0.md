## What shipped

### kailash 2.13.3 — auth Rule 2 cleanup catch-up (#779)

Patch release cutting PyPI for the auth-stub Rule 2 cleanup that landed in
PR #779 (commit `9364027b`, merged 2026-05-01) without a same-PR version
bump per `rules/build-repo-release-discipline.md` Rule 5. The bump catches
PyPI up to main; no new code.

Recap of #779 (originally landed in `kailash 2.13.2` codebase, never
released): 7 `raise NotImplementedError` stubs eliminated across
`src/kailash/{nodes,middleware}/auth/` per `rules/zero-tolerance.md`
Rule 2 § "Fake dispatch" + `rules/orphan-detection.md` Rule 3.
Default-config users previously hit `NotImplementedError` from documented
public APIs (`SSOAuthenticationNode(provider="saml")`,
`EnterpriseAuthProviderNode(method="passwordless")`, the unconditional
`_assess_behavior_risk` call). Each surface now either raises typed
`ValueError` naming the override path, returns a documented no-op default,
or — for orphan helpers with zero production callers — was deleted outright.

Locked by `tests/regression/test_auth_stub_rule2_cleanup.py` (11 tests).

### kailash-kaizen 2.16.0 — LlmDeployment four-issue parity wave (#761/#762/#763/#764)

Two-minor-version jump combining PR #780 (#761/#762) + PR #783 (#763/#764).
Cross-SDK parity with kailash-rs PR #722 / PR #724 / PR #725 / PR #726.

**#761/#762 — `openai_compatible` + `anthropic_compatible` escape-hatch presets** (from 2.15.0, never released): wraps an arbitrary HTTPS endpoint with the canonical OpenAI Chat / Anthropic Messages wire protocol — useful for vLLM, llama.cpp, LM Studio, LiteLLM proxies, OpenRouter Anthropic mode. SSRF guard runs on `Endpoint.base_url` automatically via the field validator.

**`LlmDeployment.preset_name: Optional[str]` field** (from 2.15.0): canonical preset literal exposed on every constructed deployment. Set by all 26 preset factories. The literal — NOT the host or any caller-supplied URL fragment — prevents log-aggregator label cardinality blow-up per `rules/observability.md` § 8 (schema-revealing field names).

**#763 — `LlmDeployment.supports() -> dict[str, bool]` capability matrix** (new in 2.16.0): five-key dict (`tools`, `vision`, `batch`, `caching`, `audio`) keyed off `preset_name`. Per-preset rows byte-identical to kailash-rs `CapabilityMatrix::for_preset`. Fail-closed default — unknown / future preset names AND manual `preset_name=None` constructions return all-False (structural defense — adding a new preset constructor without wiring its capability row leaves the deployment marked uncapable until the wiring lands). Returned dicts are independent copies; mutating one cannot corrupt the matrix table or another caller's result.

**#764 — `LlmDeployment.register_bedrock_region(region)` runtime override** (new in 2.16.0): process-local registry of additional Bedrock regions beyond the static `BEDROCK_SUPPORTED_REGIONS` allowlist. Hatch-of-last-resort for operators on a newly-published AWS region not yet in the kailash-py release. Format-validated against `^[a-z]{2,3}-[a-z]+-\d+$` (byte-identical regex to kailash-rs); malformed input raises new typed `InvalidRegionFormat`, distinct from `RegionNotAllowed` (well-formed-but-unknown). Idempotent; thread-safe (copy-on-write `frozenset` swap under `threading.RLock`; readers lock-free); process-local. Static allowlist short-circuits first.

Kailash-rs gates `register_bedrock_region` behind `cargo feature = "bedrock-region-override"` (default OFF). Python is single-feature-set so the function exists unconditionally; the call site IS the explicit opt-in.

## Cross-SDK

- `_PRESET_CAPABILITIES` rows byte-match kailash-rs `CapabilityMatrix::for_preset` per `rules/cross-sdk-inspection.md` § 3a (verified by reviewer's parametrized 19-preset sweep).
- `BEDROCK_SUPPORTED_REGIONS` byte-identical to kailash-rs `BEDROCK_REGIONS`.
- `_BEDROCK_REGION_RE` regex byte-identical.
- Minor non-blocking divergence noted: kailash-rs accepts `"ollama_default"` alias; Python returns all-False because no Python preset factory ever sets `preset_name="ollama_default"`. Reachable only via manual construction — fail-closed is correct behavior in the gap.
