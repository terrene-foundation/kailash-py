## What ships

- **`kailash.infrastructure.history_store`** (NEW module, ~895 LOC):
  - `WorkflowHistoryStore` (ABC), `PostgresHistoryStore`, `SQLiteHistoryStore`, `DowngradeRefusedError`
  - Schema: `workflow_runs` (run_id PK, tenant_id NOT NULL, …) + `workflow_run_events` (id PK, run_id FK, event_seq, …) with 4 tenant-scoped indexes
  - Write-time redaction via `redact_event_for_persistence`
  - 30-day retention TTL on `terminal_at`; `delete_runs_older_than(*, force_downgrade=True)` destructive-confirmation gate per `rules/schema-migration.md` Rule 7
  - Per-tenant cap (default 10,000) with WARN log on oldest-row eviction per `rules/observability.md` Rule 7
  - Per-run `asyncio.Lock` LRU cache (mirrors W1 `_checkpoint_locks`) closes `MAX(event_seq)+1` race under READ COMMITTED (M-1 fix)
  - `get_run_events` events fetch JOINs runs table with tenant predicate at data-fetch level (H-1 fix)

- **`LocalRuntime(history_store=...)` auto-subscribe** (`src/kailash/runtime/local.py:813-842`):
  - Accepts non-None `history_store=`, registers `record_event` against the W1 hook registry at construction
  - Typed `TypeError` guard on bad `record_event` (per `rules/zero-tolerance.md` Rule 3a)
  - `AsyncLocalRuntime` inherits via `super().__init__(**kwargs)`
