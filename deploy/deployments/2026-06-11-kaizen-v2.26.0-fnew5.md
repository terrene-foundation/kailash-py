## What shipped

FNEW-5 ‚Äî provider-config intrinsic default constants. The nine
`get_*_config()` functions in `kaizen.config.providers` no longer hold inline
hardcoded `default_model` literals. Each provider's final fallback is now a
documented module-level `DEFAULT_<PROVIDER>_MODEL` constant:

- **Provider-intrinsic** ‚Äî the caller has already chosen the provider, so the
  default carries no lock-in.
- **Overridable** via `KAIZEN_<PROVIDER>_MODEL`.
- **Deliberately NOT chained** to the provider-agnostic `KAIZEN_DEFAULT_MODEL`
  (chaining would reintroduce the 2.25.0 provider/model mismatch ‚Äî e.g. a
  `claude-*` model returned under `provider="openai"`).
- Zero-config `auto_detect_provider()` unchanged.

**User-visible default refresh (non-breaking):** the stale Anthropic default
`claude-3-haiku-20240307` is refreshed to `claude-haiku-4-5`. Callers using the
bare `get_anthropic_config()` default (no `model=` arg AND no
`KAIZEN_ANTHROPIC_MODEL` env) now receive `claude-haiku-4-5`. Signature and
resolution order unchanged; only the final-fallback value moved ‚Üí minor bump.

New regression suite
`tests/regression/test_issue_fnew5_provider_intrinsic_defaults.py` locks default
resolution, `KAIZEN_<PROVIDER>_MODEL` / `model=` precedence, and the
`KAIZEN_DEFAULT_MODEL` no-leak invariant across all nine providers.
