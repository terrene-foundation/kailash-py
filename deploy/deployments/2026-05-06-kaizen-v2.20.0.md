## Changes (consolidated `## [Unreleased]` → `## [2.20.0]`)

### #829 — PR #836 (commit `ba476b88`) — LLM-first trait derivation

Replaces the keyword-classifier in `Kaizen._generate_role_based_traits`
(`framework.py:513-536` pre-fix) with a `Signature`-driven LLM call. The
prior implementation was a direct violation of `rules/agent-reasoning.md`
Rule 1 (BLOCKED hardcoded classification of agent input) — `if any(word in
role_lower for word in ["research", ...])` style buckets.

**Implementation:**

- New `kaizen/core/_role_traits_signature.py` declares
  `RoleToTraitsSignature(role: str → traits_csv: str)` — private, not
  exported in `kaizen/__init__.py::__all__`.
- `framework.py:_generate_role_based_traits` now constructs a `BaseAgent`
  wrapped around `RoleToTraitsSignature`, invokes `BaseAgent.run(role=role)`
  with `temperature=0`, parses the comma-separated `traits_csv` output,
  caches per-instance.
- `Kaizen.__init__` adds `self._trait_cache: OrderedDict[str, List[str]] = OrderedDict()`
  - `self._TRAIT_CACHE_MAX = 256` — bounded LRU cache.

**Security hardening (in same PR per security-reviewer findings):**

- Output sanitization regex `^[a-z0-9_ ]{1,32}$` rejects malformed traits
  (prompt-injection defense — protects the downstream
  `_generate_role_based_prompt` which renders traits into the agent's
  system prompt).
- LRU cache via `OrderedDict.popitem(last=False)`, capped at 256
  entries (DoS defense via unique-role pollution).
- Hashed role logging at WARN — `role_hash=<sha256[:8]>` and
  `raw_len=<int>` instead of raw role string and raw output (per
  `rules/observability.md` Rule 8 spirit; PII-leakage defense).

**Failure mode change:**

- Previous: silent default-trait list returned for unmatched roles
  (Rule-1-violating fallback).
- New: `RuntimeError` when `KAIZEN_DEFAULT_MODEL` unset OR LLM call
  fails. Message names both escape hatches: pass `behavior_traits=...` in
  config OR configure `.env` provider key.

  tests for cache-hit + whitespace/case normalization.
