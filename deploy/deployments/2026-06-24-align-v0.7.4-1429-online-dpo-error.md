## kailash-align 0.7.4 — #1429 (OnlineDPOAdapter informative error)

0.7.3 (#1426) de-registered `online_dpo` from `METHOD_REGISTRY` because trl

> =1.0 removed `OnlineDPOTrainer`/`OnlineDPOConfig` upstream. But
> `OnlineDPOAdapter.build()` still called `get_method("online_dpo")` outside its
> guard, so it surfaced an opaque `AlignmentError: Unknown training method
'online_dpo'` — telling the user nothing about WHY online-dpo vanished or what
> to use instead.

- **`build()` raises an informative `TrainingError`** byte-identical to the one
  `OnlineDPOConfig.to_trl_config()` already raises in `config.py` ("Online DPO
  is unavailable: trl >=1.0 removed … use `method='dpo'` or `method='grpo'`").
  A parity regression test pins the two messages byte-identical so a future
  edit to one without the other fails loudly.
- **`learn()` surfaces the same error** — it builds the trainer first; the
  unreachable post-build trainer-construction body (constructing a trl trainer
  that can never exist) was dead code and is removed. Behavior-narrowing fix
  only; no public dataclass-signature break. `OnlineDPOConfig`/`OnlineDPOAdapter`
  remain exported back-compat surfaces.
