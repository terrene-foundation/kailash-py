## What shipped

F31 Wave 4 — the final honesty gap in the RAG-coverage workstream (brief: _"provably
correct, not merely importable"_). Five `kaizen.nodes.rag` nodes advertised ML
capabilities they only simulate; each is now honest.

**Deprecated** (warn-on-construction shim, kept importable per `zero-tolerance.md`
Rule 6a; removal scheduled next minor = 2.28.0):

- `VisualQuestionAnsweringNode` — keyword-derived placeholder answer + fixed confidence;
  never reads pixels. No VQA backend implemented.
- `ImageTextMatchingNode` — keyword-derived placeholder match scores; not CLIP/ALIGN.

**Reduced** (real core kept; docstrings corrected to stop advertising the simulated
backend; runtime behaviour unchanged):

- `MultimodalRAGNode` — real LLM query-analysis + response-generation over a hash/lexical
  placeholder encoder (not CLIP/BLIP-2). Also fixed a hardcoded `model="gpt-4-vision"` →
  env-resolved `OPENAI_VISION_MODEL` (`env-models.md` compliance).
- `FederatedRAGNode` — in-process aggregation pattern, not real distributed/network federation.
- `ColBERTRetrievalNode` — lexical late-interaction (token-overlap MaxSim) approximation,
  not a learned BERT model. Deleted the dead, never-called `_create_workflow` whose
  `token_embedder` generated random `np.random.randn(768)` embeddings; its orphaned
  None-content regression test was ported to a behavioural `run()` test.
