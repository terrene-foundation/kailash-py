## Changes (consolidated [Unreleased] → [2.19.0])

### #822 Shard 1 — PR #825 (commit `1abf940f`) — Optional/None pyright cascade + signature_programming gate

Type-only fix at `core/agents.py` cleared 26 pyright errors via `setattr` for the
`AgentTeam` attach-site (avoiding both `# type: ignore[attr-defined]` and `# pyright:
ignore[reportAttributeAccessIssue]`). Restored `signature_programming` behavior that
was silently no-op'ing because of the typing cascade.

### #822 Shard 2 — PR #826 (commit `98220cd8`) — Dead MCP integration surface deletion

Net `-1155 LOC` deletion across `core/agents.py` (598 lines), `core/framework.py` (203
lines), and 305 lines of test-file deletions for non-existent symbols. Added a 78-line
LOC invariant regression test (`tests/regression/test_issue_822_loc_invariant.py`) per
`rules/refactor-invariants.md` Rule 1 to prevent silent re-inlining.

**Surfaces removed (12 methods + 1 property):**

- `kaizen.core.agents.Agent` (CoreAgent): `expose_as_mcp_server`, `expose_as_mcp_tool`,
  `get_mcp_tool_registry`, `execute_mcp_tool`, `connect_to_mcp_servers`, `call_mcp_tool`,
  `_call_mcp_tool` (private), `_discover_servers` (private)
- `kaizen.core.framework.Kaizen`: `mcp_registry` property, `expose_agent_as_mcp_tool`,
  `list_mcp_tools`, `discover_mcp_tools`

**Why deletion (not deprecation):** all 12 methods imported `..mcp.registry::get_global_registry`,
`..mcp::AutoDiscovery`, or `..mcp::MCPConnection` — none of which have ever existed in
the kaizen source tree at any commit (`git log --oneline --all -- 'packages/kailash-kaizen/src/kaizen/mcp/registry.py'`
returns empty; `kaizen/mcp/__init__.py::__all__` has been `["EnterpriseFeatures", "MCPServerConfig"]`
since `b553104c` — the original `apps/`→`packages/` move). Each method was wrapped in
`try/except ImportError` (or `try/except Exception:`) blocks that ALWAYS fell through,
returning `None`, `[]`, or error dicts. Per `rules/zero-tolerance.md` Rule 2 (no fake
integration on documented public API) and `rules/orphan-detection.md` Rule 3 (removed-not-deprecated),
deletion is the only valid disposition.

**Migration:** the canonical user surface `kaizen.Agent` resolves to `kaizen_agents.Agent`
(per ADR-020) which never had these methods — blast radius is scoped to direct
`from kaizen.core.agents import Agent` callers. Targets: `kailash-mcp` package directly
for external MCP server/tool exposure; `kaizen.mcp.catalog_server.MemoryRegistry` for
in-process registry.

### #814 Shard 2 — already shipped on PyPI as 2.18.2 (commit `24446c7c`) — research/web-search extras
