## Done-gate finding

The post-publish clean-venv install verification caught a real defect in 2.25.0:
`import kaizen_agents.patterns` (and any import of `kaizen.core.autonomy.hooks`, e.g. for
`HookManager`) raised `ModuleNotFoundError` on a slim-core `pip install`. The hook package
eagerly re-exported `LoggingHook`/`MetricsHook`/`TracingHook`, which import
`structlog`/`prometheus-client`/`opentelemetry` (all `[observability]`-extra deps, not core)
at module scope. This was the exact slim-core import contract kaizen-agents 0.9.9 claimed to
complete — the FU5 fix addressed type-annotation imports but missed the hooks path.
