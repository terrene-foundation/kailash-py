# Deployment — kailash-nexus 2.8.0 (2026-05-31)

Single-package minor release closing **issue #1174** (Nexus FastAPI parity) — the
final acceptance criterion (AC 7: migration guide + version owner). Shards 1–4
(PRs #1210/#1211/#1212/#1213) shipped the extractor / `dependency_overrides` /
SSE / WebSocket-callback surface to `main` at 2.7.0; this release (PR #1214,
Shard 5) adds the FastAPI→Nexus migration guide and bumps the package so the full
feature set is publishable.

## Surface released (all backwards-compatible — no source removed)

- `nexus.extractors` sub-module — `Depends`, `Request`, `UploadFile`, `Multipart`,
  `Bytes`, `Headers`, `NexusHandlerError`, `DependencyOverrideMap`,
  `DependencyOverrideRuntimeMutationError` (#1174 ACs 1–4).
- `Nexus.handler_extract(name, func, ...)` — annotation-driven per-handler resolver
  chain (#1174 ACs 1–2).
- `Nexus.dependency_overrides` (`DependencyOverrideMap`) test-injection surface (#1174 AC 3).
- `Nexus.register_sse(path, on_subscribe, ...)` SSE primitive; `register_sse_endpoint`
  is now a shim over it (#1174 AC 5).
- `Nexus.register_websocket` callback overload (`on_message`/`on_connect`/`on_disconnect`/
  `allowed_origins`/`subprotocols`/`dependencies`) alongside the existing class path (#1174 AC 6).
- `packages/kailash-nexus/docs/migration-fastapi.md` migration guide (#1174 AC 7).

Deferred to follow-up (documented in guide §9, NOT shipped): `Body[T]` typed-model
extraction, `Query` extractor, cross-transport `NexusRequest`, OpenAPI generation.
