# 2026-06-01 — kailash 2.28.3 (#1200) + kailash-dataflow 2.10.1 (#1207)

Two-package patch release closing two externally-reported correctness bugs.

## kailash 2.28.2 → 2.28.3 — runtime contextvars propagation (#1200)

`AsyncLocalRuntime`, `LocalRuntime`, `ParallelRuntime`, and `ParallelCyclicRuntime`
dropped the caller's `contextvars.Context` when dispatching sync node execution
across a thread boundary (`loop.run_in_executor`, raw `threading.Thread`,
`ThreadPoolExecutor.submit`). A `ContextVar` set before `execute()` /
`execute_workflow_async()` was invisible inside a node's `run()` — the node saw
the variable's default, unlike the stdlib `asyncio.to_thread` convention.

Fix: snapshot `contextvars.copy_context()` in the caller frame and dispatch the
thread-boundary callable through `ctx.run(...)` at all six affected sites
(`async_local.py` ×2 `run_in_executor`, `local.py` raw `threading.Thread` +
sync-in-async `ThreadPoolExecutor`, `parallel.py` `run_in_executor`,
`parallel_cyclic.py` `submit` via a fresh `caller_ctx.copy()` per concurrent
dispatch so a single `Context` is never entered concurrently). The
distributed-worker path (`distributed.py` :1380 dequeue, :1489
queue-reconstructed workflow) is intentionally excluded — the workflow is rebuilt
from a Redis-queued task in a separate worker process, so the caller's context is
already gone across the queue boundary.

## kailash-dataflow 2.10.0 → 2.10.1 — Optional[list]/Optional[dict] JSONB round-trip (#1207)

`db.express.create` / `db.express.read` returned a JSON-encoded `str` instead of
the deserialized Python `list`/`dict` for model fields declared `Optional[list]`
/ `Optional[dict]` (the idiomatic nullable-JSONB-column annotation). The DB write
path was correct (JSONB array stored); the read path leaked the raw string, and a
downstream `result["tags"][0]["label"]` then hit `TypeError: string indices must
be integers`.

Root cause: `_deserialize_json_fields` (`core/nodes.py`) checked
`field_type in (dict, list)` against the raw declared annotation; for
`Optional[list]` the annotation is `Union[list, None]` (or PEP 604 `list | None`,
a `types.UnionType`), so the membership test was `False`, `json.loads` was
skipped, and the raw string was returned. Fix: new helper `_unwrap_optional_type`
collapses single-non-None-arg unions (`Optional[T]`, `Union[T, None]`, PEP 604
`T | None`) to bare `T` before the membership check; routed through the single
read/create deserialization chokepoint (`_deserialize_json_fields`, covering all
13 call sites) AND `convert_datetime_fields` (same `__origin__ is Union` blindness
affected nullable `datetime | None`). Multi-arg unions and non-union annotations
pass through unchanged.
