# detectStateFileMutationSegmentAware audit fixtures

Per `rules/cc-artifacts.md` Rule 9 + `rules/hook-output-discipline.md` MUST-4. Each fixture pins one scope-restriction predicate `detectStateFileMutationSegmentAware(command, pathRx)` (and the `detectStateFileMutation` Layer-1 matcher it calls) relies on. Inputs are bash command strings (paired with a path regex the caller passes — typically the trust-posture `STATE_PATH_RX`); expected outputs are the JSON returned by the detector — `null` (no flag) or a structural detection object naming the layer (1: redirect/tee/sed-i, 2: cp/mv/rm/chmod, 3: interpreter -c body).

The detector wraps `detectStateFileMutation` with quote-aware SEGMENT splitting so the `git commit … -m/-F` body exception applies PER SEGMENT. For a commit segment it MASKS the quoted message body (rather than skipping) before detection; because `$(…)`/backtick command-substitution AND bash-5.3 `${ …;}`/`${| …;}` funsubs EXECUTE inside double quotes, and `$'…'` ANSI-C quoting desyncs the quote scan, a commit segment containing any of those (`EXECUTES_INSIDE_QUOTES_RX`) additionally fails closed by re-scanning the RAW segment. The commit-body recognizer accepts `-m`/attached-`-m`/`-am`/`--message`/`-F`/`--file` forms. The Layer-1 redirect matcher recognizes every file-writing form — `>`, `>>`, `>|` (force-clobber), `&>`, `&>>`, and fd-prefixed `N>`/`N>>`/`N>|` — while excluding fd-dups (`2>&1`, `>&2`). Closes issue #745 Evasion 1, the redirect-on-commit sibling, and the redteam-surfaced F1 (command-sub in body) / F2 (ANSI-C desync) / F3 (`-am` false positive) / Finding-1 (redirect-operator coverage) / Finding-2 (funsub). Detector source: `.claude/hooks/lib/violation-patterns.js::detectStateFileMutationSegmentAware` + `::detectStateFileMutation`.
