# detectRepoScopeDriftBash audit fixtures

Per `rules/cc-artifacts.md` Rule 9 + `rules/hook-output-discipline.md` MUST-4. Each fixture pins one scope-restriction predicate the detector relies on. Inputs are Bash command strings; expected outputs are the JSON returned by `detectRepoScopeDriftBash(input, cwd)` — `null` (no flag) or a violation object.

| Fixture                         | Expects           | Predicate locked                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------------------- | ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `clean-current-repo.txt`        | `null`            | `gh ... --repo X` where X contains cwd basename → in-scope, no flag                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `flag-explicit-other-repo.txt`  | `halt-and-report` | `gh ... --repo X` where X is a SYNTHETIC, never-authorized off-repo string (`other-org/other-repo`) → flag with halt-and-report severity (NOT block, per MUST-2). MUST be a synthetic slug: the detector's condition-4 receipt allowance (`hasCrossRepoAuthorizationReceipt`) returns null when a REAL cross-repo-authorization receipt for the target exists on disk in `.claude/cross-repo-authz/`, so a real repo slug (e.g. a genuine sibling repo) would be suppressed the moment a real session authorizes a read/write to it — a slug that can never have a real remote/receipt keeps the flag fixture deterministic. |
| `skip-shell-variable.txt`       | `null`            | `gh ... --repo "$REPO"` — pre-expansion shell variable, detector cannot evaluate, MUST skip (MUST-3)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `skip-command-substitution.txt` | `null`            | `gh ... --repo $(...)` — command substitution, detector cannot evaluate, MUST skip (MUST-3)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `skip-braced-variable.txt`      | `null`            | `gh ... --repo "${REPO}"` — braced shell variable, MUST skip (MUST-3)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `skip-embedded-in-echo.txt`     | `null`            | `echo '...gh ... --repo X...'` — `gh` embedded as a substring inside an echo / JSON payload (preceded by a quote, NOT a command-segment boundary). The SEGMENT-ANCHORED regex `(?:^\|[;&\|\n(])\s*gh` MUST skip so the guide-first PreToolUse ceremony does not halt on a command that merely MENTIONS a cross-repo gh invocation.                                                                                                                                                                                                                                                                                           |
| `skip-embedded-in-grep.txt`     | `null`            | `grep -rn "gh ... --repo X" ...` — same substring-embedding class; `gh` follows a quote, not a segment boundary → MUST skip.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |

The cwd for every fixture is `/Users/esperie/repos/loom` (basename `loom`). `other-org/other-repo` is the canonical SYNTHETIC off-repo string for the FLAG case (it can never resolve to a real remote or accumulate a real authorization receipt, so the flag stays deterministic — see the `flag-explicit-other-repo.txt` row). The two `skip-embedded-*` fixtures deliberately keep a realistic-looking slug inside the echo/grep payload, since they assert `null` via the segment-anchor skip regardless of any receipt. The segment-anchored regex (`gh` MUST lead a command segment — start-of-string OR after `;` `&` `|` newline `(`) is what distinguishes a real cross-repo invocation from one embedded as a substring; the two `skip-embedded-*` fixtures pin that boundary.
