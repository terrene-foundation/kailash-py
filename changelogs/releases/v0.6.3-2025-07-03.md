# Release v0.6.3 - 2025-07-03

## Critical MCP Namespace Collision Fix

This release resolves a critical namespace collision issue that was preventing proper FastMCP integration in the Kailash SDK.

## 🐛 Fixed

### MCP Namespace Collision (Critical)
- **Issue**: Local `kailash.mcp` package was shadowing external `mcp.server` package, causing `ModuleNotFoundError: No module named 'mcp.server.fastmcp'`
- **Solution**: Renamed `src/kailash/mcp` to `src/kailash/mcp_server` to eliminate namespace collision
- **Impact**: FastMCP imports now work correctly in all contexts

### API Design Consolidation
- **Eliminated redundant server implementations**:
  - `MCPServerBase`: Abstract base class for custom implementations (e.g., AIRegistryServer)
  - `MCPServer`: Main concrete server with all production features (enhanced capabilities)
  - `SimpleMCPServer`: Alias for backward compatibility
  - `EnhancedMCPServer`: Alias for backward compatibility
- **Clear inheritance hierarchy** for different use cases
- **Developer-friendly naming** that matches expectations

### Import Path Updates
- Updated all imports across the codebase from `kailash.mcp` to `kailash.mcp_server`
- Fixed FastMCP import path from `mcp.server.fastmcp` to `mcp.server`
- Updated 25+ files with proper import references

### Zero Skipped Tests Policy

- **Added proper mocking** for FastMCP components in unit tests
- **100% test execution** - no skipped tests remaining

## 🔧 Technical Details

### Root Cause Analysis
The namespace collision occurred because Python's import system prioritized the local `kailash.mcp` package over the external `mcp` package when resolving `mcp.server` imports. This caused the FastMCP framework to be unreachable.

### Solution Architecture
Renaming to `kailash.mcp_server` creates a clear namespace separation while maintaining all functionality. The consolidated server design provides a cleaner API surface with backward compatibility.
