# Unreleased Changes

This directory contains changelog entries for features that haven't been released yet.

Each file should be named with a descriptive identifier and contain the changes in the Keep a Changelog format.

## Format

Create files with meaningful names like:
- `feature-name.md`
- `bugfix-description.md`
- `enhancement-title.md`

Each file should contain sections like:
```markdown
### Added
- Description of what was added

### Changed
- Description of what was changed

### Fixed
- Description of what was fixed
```
