## Wave-B FINDINGS (genuine divergences the cutover MUST decide on)

### Finding #1 — Legacy Anthropic tool-calling is BROKEN; four-axis fixes it

`providers/llm/anthropic.py::AnthropicProvider.chat` (1) never adds `tools` to
`create_kwargs` (ignores the tools argument) and (2) hardcodes
`response.content[0].text` + `tool_calls: []`, so a real `tool_use` response
raises `AttributeError: 'ToolUseBlock' object has no attribute 'text'`. The
four-axis `anthropic_messages` wire emits tools AND parses `tool_use` correctly.

**Disposition:** migrating Anthropic consumers is a behavior IMPROVEMENT
(tool-calling starts working), NOT parity-neutral. The dual-run shadow would
correctly log this as a divergence. Legacy is deleted in Wave-C, so the legacy
bug is documented, not fixed. **No action needed for Wave-B beyond awareness** —
any Anthropic consumer relying on the (broken) legacy tool path was already
getting `tool_calls: []`.

### Finding #2 — Google finish_reason casing divergence

Legacy `GoogleGeminiProvider.chat` lowercases the Gemini `finishReason`
(`STOP` → `stop`); the four-axis `google_generate_content` wire preserves the raw
provider value (`STOP`). A consumer testing `finish_reason == "stop"` breaks for
Gemini on four-axis.

**Disposition (needs Wave-B decision):** either (a) normalize the four-axis
Google wire to lowercase finish_reason for behavior-neutral migration, OR (b)
accept the raw-value change and update consumers. **Recommend (a)** — a
behavior-neutral migration is the safer default; the raw-value change buys
nothing a consumer asked for. Low effort (one map in the google wire's
`parse_response`). Pinned by `test_google_parse_parity_documents_finish_reason_casing_delta`.

## Wave-B BLOCKER

### azure_ai_foundry has NO four-axis wire

The legacy `azure_ai_foundry` provider has no four-axis equivalent; the shared
resolver raises a typed `UnsupportedDeploymentProvider` (loud, not a silent
fallback). **Any azure_ai_foundry consumer cutover is BLOCKED until a four-axis
azure_ai_foundry wire exists.** Enumerate azure_ai_foundry consumers before Wave-B
and either build the wire or scope them out of the cutover.

## One-sided by construction (no migration risk)

Bedrock / Vertex / Mistral are four-axis-only wires (no legacy provider) — NEW
capabilities, not migrations. Zero Wave-B cutover risk. Pinned structurally by
`test_bedrock_vertex_mistral_are_fouraxis_only_one_sided`.
