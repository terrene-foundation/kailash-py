## Verified PASSED checks

1. **Protection wired on every Express mutation surface** — `_check_protection_if_enabled` called in `create` (l. 664), `update` (l. 855), `delete` (l. 948), `upsert` (l. 1283), `upsert_advanced` (l. 1371), `bulk_create` (l. 1449), `bulk_delete` (l. 1598), `bulk_upsert` (l. 1678). `bulk_update` correctly delegates via `self.update()` (l. 1535) and re-raises `ProtectionViolation` (l. 1537), not folding it into `failed_count` — Shard 1 hard-stop discipline holds.
2. **#1058 Shard 3 UPSERT enum gap closed** — `OperationType.UPSERT = "upsert"` added (`core/protection.py:37`); `_operation_mapping["upsert"] = OperationType.UPSERT` (`core/protection.py:344`); single-record upsert is now first-class, no longer falls through to CUSTOM_QUERY. `count` → `OperationType.READ` (`core/protection.py:327`) closes the count over-block under read-only. The classmethod read-only defaults stay `{OperationType.READ}` so UPSERT is correctly excluded from read-only sets.
3. **#1058 Shard 4 import_file swallow closed** — `import_file` re-raises `ProtectionViolation` on BOTH `upsert` (l. 2006) and `bulk_create` (l. 2023) branches BEFORE the generic `except Exception` (l. 2017, 2027). Tested re-raise discipline matches `bulk_update` Shard 1.
4. **#1058 Shard 2 single-check + sentinel discipline** — Express sets `node._express_protection_precheck_done = True` before calling `node.async_run()` on every mutation path (l. 672, 863, 953, 1286, 1375). Inner check in `protection_middleware.py:351` honors the sentinel and skips the duplicate `check_operation` — no double `auditor.log_allowed` rows on the happy path.
5. **Protection check fires BEFORE field validators** — `_check_protection_if_enabled` precedes `_validate_if_enabled` on `create` (l. 661 → 664 → 666), `update` (l. 855 → 858), `upsert` (l. 1283 → 1284). Defense-in-depth gap closed.
6. **`ProtectionViolation` survives `AsyncNode.execute_async` re-raise allowlist** — `core/protection.py:166-203` subclasses `NodeExecutionError` (not bare `Exception`) per issue #1050 Shard 1a. Workflow-runtime hot path preserves the typed violation.
7. **Sanitizer contract — token-replace (Rule 1)** — `nodes.py:849-869` token-replaces `; DROP|DELETE|INSERT|UPDATE|EXEC` → `STATEMENT_BLOCKED`, `UNION SELECT` → `UNION_SELECT`, `DROP TABLE` → `DROP_TABLE`, `DELETE FROM` → `DELETE_FROM`, etc. NOT quote-escape. Grep-able audit trail intact.
8. **Sanitizer contract — type-confusion raise (Rule 2)** — `nodes.py:940-948` raises `ValueError("parameter type mismatch: ...")` for `str`-declared field receiving `(dict, list, set, tuple)`. Same gate on the bulk path (`nodes.py:1005-1013`). The sanitizer itself returns `dict`/`list`/`set`/`tuple` UNCHANGED through `safe_types` (`nodes.py:824-840`) so the gate sees the real container type — Rule 2 is enforced AT the gate, not pre-coerced away. Comment on l. 805-823 documents the load-bearing-passthrough rationale.
9. **Sanitizer contract — dict/list returned as-is for declared dict/list fields (Rule 3)** — `safe_types` tuple at `nodes.py:824-836` includes `dict, list, set, tuple`. Bug #515 (premature `json.dumps()` on dict/list breaking param binding) is structurally prevented.
10. **DEFENSE-2 tests cover both Sanitizer Rule 1 + Rule 2** — `tests/unit/security/test_sanitizer_public_api.py` exercises CreateNode through the same `express._create_node("Model", "Create")` construction path Express uses.
