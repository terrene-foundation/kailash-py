## Affected API

Package import surface: `kailash.runtime`, `kailash.dataflow`, `kailash.kaizen`, `kailash.pact`, `kailash.kaizen_agents`, `kailash.kailash_align` — i.e. every public sub-package's production source tree under `src/` and `packages/*/src/`.

Specifically the violation is at the source-comment surface, not at the runtime API; nothing user-visible breaks today, but the documented zero-tolerance rule against `TODO/FIXME/HACK/STUB/XXX` markers in production code is being silently violated.

## Minimal repro

```bash
# Run from a fresh checkout of terrene-foundation/kailash-py at main
$ git rev-parse HEAD                                      # whichever main is at
$ grep -rnE 'TODO-(0[0-9]+|1[0-9]+|2[0-9]+)' src/ packages/*/src/ \
    | grep -vE ':\s*///|:\s*//!|/build/' | wc -l
244

$ grep -rlnE 'TODO-(0[0-9]+|1[0-9]+|2[0-9]+)' src/ packages/*/src/ \
    | grep -vE '/build/' | wc -l
118

# Distinct tracker IDs across the codebase
$ grep -rohE 'TODO-(0[0-9]+|1[0-9]+|2[0-9]+)' src/ packages/*/src/ \
    | sort -u | wc -l
98

# Top-5 production-source files by tag density:
#   28 packages/kaizen-agents/src/kaizen_agents/agents/autonomous/base.py
#   15 src/kailash/runtime/local.py
#    8 packages/kailash-dataflow/src/dataflow/migrations/integrated_risk_assessment_system.py

#    6 packages/kailash-kaizen/src/kaizen/tools/native/__init__.py
```

A spot-check shows three classes of marker:

1. **Section header / breadcrumb to completed work.** Examples like `# === Coordinated Shutdown (v0.12.0, TODO-015) ===` in `runtime/local.py` describe code that ships and works; the `TODO-` prefix is a vestigial tracker tag, not an unfinished-work signal.
2. **Active iterative TODO.** Examples like `# TODO-153: Type-Aware Field Processor` in `dataflow/__init__.py` reference forthcoming work; per zero-tolerance Rule 6 these are permitted ONLY when actively tracked (workspace todos / GH issue links).
3. **Forwarded reference to an external tracker.** Examples like `# Integration with existing migration components (TODO-137,138,140,142)` reference cross-component dependencies; whether these are still active is unverifiable from the comment alone.

## Expected vs actual

**Expected** (per the SDK's own published `rules/zero-tolerance.md` Rule 2): production code MUST NOT contain `TODO`, `FIXME`, `HACK`, `STUB`, `XXX` markers. Rule 6 carves out an exception for "iterative TODOs" only when the work is actively tracked; section-header / completed-work breadcrumbs are not iterative work and do not qualify for the exception.

**Actual**: 244 marker hits across 118 production-source files. A subset are completed-work breadcrumbs (already shipped — equivalent of the `(GOV-NNN)`-style parenthetical tags the codebase uses elsewhere); a subset are iterative TODOs whose tracking link is non-obvious from the comment; a subset reference cross-component dependencies whose activity status is unclear.

## Severity

**LOW** — no runtime behavior is broken; this is a documentation-discipline / audit-hygiene finding.

## Acceptance criteria

- [ ] Audit each of the 244 production-source markers and classify into one of: `completed-work breadcrumb` / `active iterative TODO with tracker link` / `obsolete tracker, no longer relevant`.
- [ ] For `completed-work breadcrumb`: rename the `TODO-NNN` prefix to a non-keyword form (e.g. `(V<release>-NNN)` parenthetical, matching the existing `(GOV-NNN)` convention used elsewhere in the codebase) so `grep TODO` audits no longer surface them.
- [ ] For `active iterative TODO`: ensure each has a same-line link to a tracking GH issue OR a `workspaces/<project>/todos/active/...` path, per Rule 6's iterative-TODO exception.
- [ ] For `obsolete tracker`: delete the comment.
- [ ] After cleanup, the canonical grep returns zero: `grep -rnE 'TODO-(0[0-9]+|1[0-9]+|2[0-9]+)' src/ packages/*/src/ | grep -vE ':\s*///|:\s*//!|/build/' | wc -l` → `0`.
- [ ] Add a CI check (or pre-commit hook) that fails on any new `TODO-NNN` introduced into production source without a same-line tracker link.
