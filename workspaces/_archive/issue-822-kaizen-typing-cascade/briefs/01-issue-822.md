# Brief — Issue #822 (kaizen Optional/None typing cascade)

Source: github.com/terrene-foundation/kailash-py/issues/822 (filed 2026-05-04 from #814 Shard 2)

## Summary (verbatim from issue)

Pre-existing pyright `reportArgumentType` warnings in `kaizen/__init__.py` and
`kaizen/core/framework.py` use the `param: str = None` lying-type-annotation
pattern (parameter accepts `None` at runtime but typed as non-Optional).
Discovered during #814 mid-implement; out-of-scope for #814 (which was scoped
to `tools/native/` + `research/`).

## Affected sites (per brief)

- `kaizen/__init__.py:127` — `def create_agent(name: str = None, config: dict = None, ...)`
- `kaizen/__init__.py:155` — `def get_resolved_config(explicit_config: dict = None) -> dict`
- `kaizen/core/framework.py:341` — `Kaizen.create_agent(agent_id: str = None, ...)`
- `kaizen/core/framework.py:344` — `Kaizen.create_agent(..., name: str = None, ...)`

## Cascade impact (per brief)

Fixing the lying types reveals additional issues in `framework.py`:

- 4 errors: `reportMissingImports` for `..mcp.registry`, `reportUndefinedVariable`
  for `Signature`, `reportOptionalCall` x2
- 6+ warnings: multiple `reportAttributeAccessIssue` on `Agent.role`,
  `Agent.expertise`, `Agent.capabilities`, `Agent.behavior_traits`,
  `Agent.authority_level`

Cascade extends into `core/agents.py` (per #814 grep).

## Severity (per brief)

LOW — warnings only, not errors. CI doesn't pyright-check the kaizen package
(only `src/kailash/`); these surface only in dev tools / IDE LSP. No runtime
impact.

## Acceptance (per brief)

- [ ] `uv run pyright kaizen/__init__.py` → 0e/0w
- [ ] `uv run pyright kaizen/core/framework.py` → 0e/0w
- [ ] `uv run pyright kaizen/core/agents.py` → 0e/0w
- [ ] All existing tests pass.
- [ ] No regression in `kaizen.create_agent(...)` runtime semantics (callers
      passing `None` still work).
